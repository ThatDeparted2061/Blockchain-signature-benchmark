# Blockchain-signature-benchmark

## Overview

This project benchmarks digital signature performance between RSA (PSS, probabilistic) and ECDSA at equivalent security levels.

**Important:**
- Only the script `benchmark_complete.py` is needed for full benchmarking.
- All results and graphs are generated by this script.

## Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install cryptography matplotlib seaborn pandas psutil
```

## Run the Benchmark

```bash
python3 benchmark_complete.py
```

- Full suite covers: 112, 128, and 192 bits (RSA and ECDSA).
- Outputs CSV: `results/benchmark_results_comprehensive.csv`
- Produces graphs saved in `results/graphs/`

## Metrics Reported

- Signing and verification time
- CPU time consumed by the benchmark process (user + system, delta per algorithm)
- Memory delta during benchmarked operations (MB, baseline-subtracted)
- Key and signature sizes
- Ratios for performance comparisons (RSA/ECDSA)

## Key Notes

- RSA signing uses RSASSA-PSS for secure, probabilistic signatures.
- No other .py benchmarking scripts are needed—`benchmark_complete.py` replaces prior scripts for clarity.
- For lab or CI runs, expect longer runtimes for ultra-large keys (RSA 15360/30720 bits).
- Verification-ratio summary is computed from measured batch verification runs per transaction count (not from constant single-op scaling).
